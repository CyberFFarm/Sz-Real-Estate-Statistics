# 深圳房地产预售数据分析系统 · 使用说明

## 文件说明

| 文件 | 作用 | 保留/删除 |
|------|------|-----------|
| `server.py` | 后端代理服务 + SQLite 数据库操作 | ✅ **必须保留** |
| `index.html` | 前端页面（数据总览/项目详情/数据分析） | ✅ **必须保留** |
| `szfdc_data.db` | SQLite 数据库（4387条预售证 + 缓存房源价格） | ✅ **必须保留** |

**核心只需要 3 个文件**：`server.py` + `index.html` + `szfdc_data.db`，放到同一个目录下即可运行。

---

## 环境要求

- Python 3.7+
- 无需安装第三方依赖（只用 Python 标准库）

---

## 使用步骤

### 1. 启动服务

```bash
cd 项目目录
python3 server.py
```

启动后显示：
```
深圳房地产数据分析系统 v3.0
==================================================
服务启动: http://localhost:8080
按 Ctrl+C 停止
```

### 2. 访问页面

浏览器打开 `http://localhost:8080`

### 3. 首次使用

- 自动加载数据库中的预售证列表（4387条）
- 点击 **💾 批量缓存房源价格** 按钮，系统自动遍历近5年项目并缓存房源价格到数据库
- 查看进度在状态栏实时显示

---

## 迁移到其他电脑/服务器

### 方式一：直接复制（推荐）

将 3 个核心文件复制到目标电脑同一目录：
```
server.py
index.html
szfdc_data.db
```
然后 `python3 server.py` 启动即可。数据库已包含所有缓存数据。

### 方式二：仅复制代码，重新构建数据库

如果不需要历史缓存数据（或数据库太大），只复制 `server.py` 和 `index.html`：

```bash
python3 server.py
# 首次启动自动拉取预售证列表
# 手动点击"批量缓存房源价格"重建价格数据库
```

### 数据缓存机制

- `szfdc_data.db` 是本地工作数据库，启动后自动创建（`setup.sh` 从 `data/` 分片组装）
- 新的房源价格实时写入 DB，不会被 API null 覆盖
- `data/` 分片是初始快照，仅用于首次部署
- **更新分片到 Git**：
  ```bash
  rm data/szfdc_data_part_*
  split -b 20m szfdc_data.db data/szfdc_data_part_
  git add data/ && git commit -m "update db snapshot" && git push
  ```

### ⚠️ 数据库说明

- **创建时间**：2026 年 6 月 3 日
- 深圳住建局公开 API 中，房源状态一旦变更为"已备案、已签认购书、已录入合同"，该房源的价格字段即被**服务端抹除**（返回 null/--）
- 因此，**2026 年 6 月 3 日之前已被认购/备案的房源，价格信息在本数据库中缺失**
- 本系统通过价格保护机制（DB 存储 + null 值覆盖保护），确保从今天起新抓取的价格不会被后续状态变更覆盖
- **后续新增的预售项目，只要在房源首次变为"已售"之前打开过一次，价格即可被完整保留**

## 启动服务

```bash
python3 server.py
```

## 数据更新

- **预售证列表**：点击仪表盘"🔄 刷新列表"重新从住建局拉取
- **房源价格**：点击"💾 批量缓存房源价格"扫描近期项目
- **单个项目**：打开任意项目详情，价格自动存入数据库
- 数据库中的价格不会被后续 API 的 null 值覆盖（价格保护机制）
